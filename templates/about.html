About Employee Management System

The Employee Management System is a comprehensive web application designed to streamline the administration of employees and their activities within an organization. This project is built using Django and provides distinct interfaces and functionalities for different user roles: Students ,Admins, and Employees.

Key Features

User Roles and Permissions:

  • Students: Can interact with the system as clients, booking items and interacting with employee services.
  • Admins: Have access to the admin dashboard, where they can manage all users, approve employee accounts, and handle administrative tasks.
  • Employees: Have access to an employee-specific dashboard where they can view and manage their bookings and personal details.

Authentication and Authorization:

  • Users must authenticate themselves to access the system.
  • Role-based access control ensures that users only access areas pertinent to their roles.

Profile Management:

  • Users can upload and update their profile pictures.
  • Personal details and account settings can be managed within the user dashboards.

Booking System:

  • Employees can book and manage items, with details such as rental periods and descriptions.
  • Admins can oversee all bookings and manage the inventory of items.

Approval Workflow:

  • New employee registrations require approval by an admin before they can fully access their account features.
  • Admins can approve or deny these registration requests within the admin interface.

Django Admin Customization:

  • Enhanced admin interface with custom fields and permissions.
  • Admins can manage user statuses, approve logins, and handle email confirmations.

Technical Overview

Frontend:

HTML, CSS, JavaScript, with jQuery for interactivity.

Backend:

Django, with a custom middleware to handle approval and role-based access control.

Database:

Utilizes Django’s ORM to manage database interactions and data persistence.

Storage:

Profile images and other media files are handled using Django’s file storage system.

Middleware

A custom middleware ensures that:

  • Only approved users can access the system.
  • Access to certain URLs is restricted based on user roles.
